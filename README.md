# 🚲 따릉이 사용 현황 분석

서울 공공자전거 **따릉이**의 대여소 위치와 2023–2025년(36개월) 이용 데이터를 결합해
공간 분포의 수요-공급 정합성을 진단하고, 미래 수요를 예측한 데이터 분석 프로젝트.

📊 **[인터랙티브 리포트 보기 → report.html](https://sciencemj.github.io/seoul-bike-analysis/report.html)**
　(로컬에서는 [`report.html`](report.html)을 브라우저로 열기 · 분석 노트북: [`seoul-bike.ipynb`](seoul-bike.ipynb))

---

## TL;DR

- 대여소 **공급은 이미 수요를 거의 선형으로 추종**한다 (자치구 거치대–수요 상관 **r = 0.92**).
- 즉 처음 잡은 *"수요-공급 mismatch"* 문제 정의는 약하다. **따릉이의 진짜 과제는 배치가 아니라 타 이동수단과의 경쟁 우위**다.
- 실제로 기후동행카드(대중교통 무제한권) 확산 후 **2025년 수요 −14.8%**(공급은 불변) — 모달 경쟁의 증거.

## 핵심 결과

| 지표 | 값 |
|---|---|
| 분석 대여소 | 2,933개 (마스터 3,230 / 거치대 38,331) |
| 연 대여건수 | 약 4,203만 건 |
| 평균 이동거리 | 2.36 km/건 · 연 탄소절감 약 22,943톤 |
| 이용자 | 남 63.6% / 여 36.4% · 20–30대 55.6% · 정기권 83.2% |
| 수요 예측 최적 모델 | `seasonal_naive` (MAPE 14.5%) > SARIMA 17.6% |
| 2024→2025 수요 변화 | **−14.8%** (일평균 102,391건) |

## 데이터

| 데이터 | 출처 | 접근 |
|---|---|---|
| 대여소 마스터 (OA-21235, `tbCycleStationInfo`) | 서울 열린데이터광장 | OpenAPI (라이브) |
| 이용정보 월별 (OA-15248) | 서울 열린데이터광장 | 반기 CSV → 집계 (`scripts/preprocess.py`) |

## 분석 방법

1. 대여소 위치 지도화 (folium)
2. 대여소별 월 이용량 시각화 (버블맵 · 자치구 공급-수요 4분면)
3. 시계열 모델 비교 백테스트 → 미래 수요 예측 (naive / seasonal_naive / Holt-Winters / SARIMA)
4. 예측 수요 평균 이상/이하 분류 → 증설·재배치 후보 도출
5. 세그먼트(성별·연령·대여구분)·사회적 효익(탄소·이동거리) 분석

## ⚠️ 한계 · 문제 재정의

이 연구의 가장 큰 한계는 데이터가 아니라 **출발점의 문제 정의**다.
분석 결과 공급≈수요(r=0.92)임이 드러나, 증설·재배치(방법 4)의 실효 효과는 제한적일 가능성이 높다.
따릉이의 다음 의사결정은 **경쟁 우위·이용자 유지**로 옮겨가야 한다.

> 근거: 백진우, 「날씨 더 따뜻, 킥보드도 줄었는데… 따릉이 이용량 15% 감소, 왜?」, 오마이뉴스, 2026-01-17.
> 기후동행카드로 대중교통이 따릉이를 대체(자동차 교통량 감소는 0.5%뿐). 본 데이터의 −14.8%와 일치.

## 🔭 추후 연구

- **A.** 공공이동수단(지하철·버스·PM) vs 따릉이 수요 비교 — 대체/보완 탄력성
- **B.** 기후동행카드 도입 효과 인과 분석 (이중차분 DiD)
- **C.** 무제한권 가입자의 따릉이 이탈 코호트 분석
- **D.** 날씨·요금 통제 후 순수 모달 경쟁효과
- **E.** 따릉이 고유 강점(first/last-mile, 운동 효용) 영역 식별
- **F.** 기후동행카드–따릉이 통합 시 수요 회복 시뮬레이션

## 재현

```bash
uv sync                        # 의존성 설치
echo 'OPENAPI=발급키' > .env    # 서울 열린데이터광장 OpenAPI 키 (대여소 마스터용)
uv run python scripts/preprocess.py   # OA-15248 다운로드 → data/processed 생성
# 분석: seoul-bike.ipynb 실행 / 결과: report.html 열기
```

> 집계 데이터(`data/processed/`)는 저장소에 포함돼 있어 대여소 마스터(OpenAPI)만 키가 필요하다.
